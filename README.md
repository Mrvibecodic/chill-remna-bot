# chill-remna-bot

Телеграм-бот «магазин + админка» для панели **Remnawave**. Продаёт доступ к VPN: пользователь выбирает тариф, оплачивает и за пару минут получает ключ. Вся настройка — прямо в Telegram: на сервере вы вписываете два значения и запускаете одну команду.

Один бинарь на Go, запуск в Docker. База данных — SQLite или PostgreSQL (бот поднимает Postgres сам). Конфигурация хранится в БД в зашифрованном виде, поэтому переживает перезапуск.

> Эта инструкция написана максимально подробно и по шагам. Если вы никогда не работали с сервером — просто выполняйте команды по порядку, копируя их целиком.

## 1. Что умеет бот

**Для пользователя:**

- Покупка подписки на 1 / 3 / 6 / 12 месяцев (сроки и цены настраиваете вы).
- **Триал** (бесплатный пробный период) — включается одной кнопкой.
- **Продление** подписки: отдельная кнопка появляется за 7 дней до конца (или сразу, если сейчас триал). Время не пересчитывается, а **добавляется** к текущему сроку.
- **Баланс (кошелёк) в рублях**: пополнить один раз и платить с баланса; есть автосписание и «планировщик расходов» (на сколько хватит суммы).
- Напоминания об окончании триала и подписки (за 7 / 3 / 1 день — настраивается).
- Красивая карточка «Подписка активна до …» с **копируемой** ссылкой и датой по МСК.

**Способы оплаты (каждый включается отдельно):**

| Метод | Что это | Валюта |
|---|---|---|
| **P2P** | Перевод на карту, вы подтверждаете оплату вручную | ₽ |
| **Telegram Stars** | Встроенная валюта Telegram ⭐ | ⭐ (≈ пересчёт из ₽) |
| **ЮKassa** | Карты/СБП через ЮKassa, вебхук с перепроверкой платежа | ₽ |
| **CryptoBot** | Оплата криптой по курсу, цены в рублях (фиат-режим) | ₽ → крипта |

**Для администратора (всё в Telegram, без правки конфигов):**

- **Тарифы**: быстрый шаблон или тонкая настройка — базовая цена, трафик (ГБ) на тариф, лимит устройств **HWID на тариф**, стратегия сброса трафика (`NO_RESET / DAY / WEEK / MONTH / MONTH_ROLLING`).
- **Сквады**: один внешний + несколько внутренних, по названиям.
- **Интерфейс**: свои баннеры на каждый раздел, премиум-эмодзи, контакты (поддержка + группа).
- **Пользователи**: карточка с активной подпиской, удаление из бота и (по запросу) из панели.
- **Платежи**: лог оплат с итогами (число пользователей и сумма).
- **Сервис**: статус панели, статистика по CPU/памяти контейнера, самообновление `/update`, поддомен, лог API-запросов, вебхуки, уведомления.

**Технические особенности:**

- Самоменеджмент Docker: бот стартует **без базы**, при выборе Postgres сам поднимает контейнер БД через `docker.sock` и переносит данные из SQLite.
- Идемпотентные платежи + фоновый **реконсилятор** (добивает оплату, если вебхук не дошёл).
- Двуязычный интерфейс RU/EN.

## 2. Что нужно перед стартом

1. **Сервер с Linux** (Ubuntu/Debian подойдут), на нём установлены **Docker** и **Docker Compose v2**. Проверка:
   ```bash
   docker version
   docker compose version
   ```
   Если команд нет — поставьте Docker по официальной инструкции: https://docs.docker.com/engine/install/

2. **Работающая панель Remnawave** (бот к ней подключается, сам её не ставит).

3. **Токен бота.** Откройте [@BotFather](https://t.me/BotFather) → команда `/newbot` → задайте имя → получите строку вида `123456789:AAEabc...`. Это `BOT_TOKEN`.

4. **Ваш Telegram ID** (число). Откройте [@userinfobot](https://t.me/userinfobot) → он пришлёт ваш `Id`, например `000000000`. Это `ADMIN_TELEGRAM_ID` — у этого аккаунта будет доступ в админку.

5. **API-токен панели Remnawave** — создаётся в дашборде панели (раздел с API-ключами, роль API). Его **не** надо вписывать в файл — введёте в мастере внутри бота.

## 3. Установка за 4 шага (Docker Compose)

Образ собирает CI и публикует в GitHub Container Registry (GHCR): `ghcr.io/mrvibecodic/chill-remna-bot:latest`. На сервере вы только берёте `docker-compose.yml`, вписываете два значения и запускаете — **никакой сборки на сервере**.

### Шаг 1. Создайте папку и возьмите docker-compose.yml

```bash
mkdir -p /opt/remnachillbot && cd /opt/remnachillbot
curl -fsSL -o docker-compose.yml \
  https://raw.githubusercontent.com/Mrvibecodic/chill-remna-bot/main/docker-compose.yml
```

### Шаг 2. Впишите два значения

Откройте `docker-compose.yml` в редакторе:

```bash
nano docker-compose.yml
```

Найдите и замените две строки (остальное не трогайте):

```yaml
      BOT_TOKEN: "123456789:AAEabc..."     # токен от @BotFather
      ADMIN_TELEGRAM_ID: "000000000"        # ваш Telegram ID (число)
```

Сохраните: в `nano` это `Ctrl+O`, `Enter`, затем выход `Ctrl+X`.

> Если ставите **не** в `/opt/remnachillbot` — поправьте в этом же файле путь `/opt/remnachillbot:/compose` и переменную `COMPOSE_HOST_DIR` на свой каталог. Если контейнер панели называется не `remnawave` — задайте `PANEL_CONTAINER` с правильным именем.

### Шаг 3. Скачайте образ и запустите

```bash
docker compose pull
docker compose up -d
```

Образ скачивается готовым из GHCR (его собрал CI) — это быстрее сборки и не грузит сервер. Посмотреть логи:

```bash
docker compose logs -f
```

(выход из просмотра логов — `Ctrl+C`, бот при этом продолжает работать).

### Шаг 4. Откройте бота и пройдите мастер

Напишите вашему боту в Telegram команду **`/start`** — запустится мастер настройки (см. следующий раздел).

**Готово.** Бот стартует без базы данных; БД и связь с панелью вы зададите в мастере.

## 4. Мастер настройки в Telegram

После `/start` (или `/setup`, если хотите пройти заново) бот по шагам задаёт вопросы — на каждом вы нажимаете кнопку или присылаете значение. Бот всё проверяет сам.

1. **Язык** — Русский / English.

2. **База данных:**
   - **SQLite** — бот создаёт файл в томе `bot-data`, больше ничего поднимать не нужно. Подходит для небольших магазинов.
   - **PostgreSQL 17** — бот сам допишет сервис `db` в `docker-compose.yml`, поднимет его через `docker.sock` и переключится без перезапуска. Если в SQLite уже были данные — они **переносятся**. Не получилось поднять Postgres — бот спокойно останется на SQLite.

3. **Где бот относительно панели:**
   - **Локально** (рекомендуется) — бот и панель на одном сервере. Бот сам подключится к docker-сети панели; адрес панели подставится как `http://remnawave:3000`.
   - **Удалённо** — панель на другом сервере.

4. **Как установлена панель** (только для «Удалённо»):
   - **Официально (Caddy)**
   - **Скрипт eGames (nginx)**

5. **Реквизиты:**
   - **URL панели** — только для «Удалённо».
   - **API-token панели** — тот, что вы создали в дашборде.
   - **Кука** — только «Удалённо» + eGames. Бот подскажет: на сервере панели в `/opt/remnawave/nginx.conf` есть блок `map $http_cookie $auth_cookie { "~*ИМЯ=ЗНАЧЕНИЕ" 1; }`. Пришлите боту строку `ИМЯ=ЗНАЧЕНИЕ`.
   - **X-API-Key** — только «Удалённо» + Caddy, если `/api` дополнительно защищён ключом.

6. **Проверка связи** — бот сам вызывает `/api/system/health` и показывает число пользователей в панели. Если число пришло — связь есть.

Всё введённое хранится в БД зашифрованным и переживает перезапуск — повторно мастер проходить не нужно.

## 5. Настройка тарифов и оплат (админка)

Откройте меню админа (вы увидите его автоматически, так как ваш ID указан в `ADMIN_TELEGRAM_ID`). Структура:

```
💳 Купить
[🎨 Интерфейс]   [💳 Подписка и оплата]
[🛠 Управление ботом]
🏠 Главная
```

### 5.1. «Подписка и оплата»

- **🪄 Быстрая настройка тарифов** — мастер задаёт цены/сроки за несколько шагов (включая трафик и HWID). Самый быстрый старт.
- **✏️ Тонкая настройка тарифа** — вручную по пунктам:
  - **Базовая цена** на каждый срок (1/3/6/12 мес).
  - **Трафик** (ГБ) на тариф; `0` = безлимит.
  - **Устройства (HWID)** на тариф — можно, например, на годовой дать больше устройств.
  - **Стратегия сброса** трафика: `NO_RESET`, `DAY`, `WEEK`, `MONTH`, `MONTH_ROLLING` (скользящие 30 дней от активации).
- **🎁 Триал** и **📡 Сквады** (внешний один + внутренние через запятую).
- **Способы оплаты** — P2P / Stars / ЮKassa / CryptoBot. Рядом с каждым ✅/❌ показывает, включён ли он.

Все цены задаются **в рублях**. P2P и ЮKassa всегда в ₽; Stars пересчитываются из ₽ приблизительно (бот это помечает); CryptoBot работает в фиат-режиме — цена в ₽, оплата криптой по курсу.

### 5.2. «Управление ботом»

Пользователи, лог платежей (с итогами), статус панели, `/update`, поддомен, лог API, вебхуки, уведомления (напоминания об окончании триала/подписки — по умолчанию включены для триала и за 3 и 1 день).

### 5.3. «Интерфейс»

Свои баннеры на каждый раздел (формат 2:1, 1280×640, чтобы Telegram не пересжимал), премиум-эмодзи, контакты. Кнопки «Поддержка» и «Группа» в меню пользователя появляются, когда заданы их ссылки в **Интерфейс → Контакты**.

## 6. Подключение платёжек: вебхуки

ЮKassa и CryptoBot подтверждают оплату **вебхуком** — HTTP-запросом на ваш сервер. Внутри контейнера бот слушает порт **`8080`**:

| Путь | Для чего |
|---|---|
| `GET /healthz` | проверка «жив ли бот» |
| `POST /webhook/yookassa` | уведомления ЮKassa |
| `POST /webhook/cryptobot` | уведомления CryptoBot |
| `POST /webhook/platega` | уведомления Platega |
| `POST /webhook/tribute` | уведомления Tribute |
| `POST /webhook/remnawave` | события панели |

**Способ A (рекомендуется) — встроенный HTTPS прямо в боте.** Бот сам получает TLS-сертификат Let's Encrypt и слушает 80/443 — без отдельного nginx/Caddy. Шаги:

1. Заведите домен/субдомен и направьте его A-записью на сервер бота (напр. `bot.example.com` → IP).
2. В админке: **Управление → Вебхуки → 🌐 Публичный HTTPS** (вкл) → **🔗 Домен** (вписать домен) → **🚀 Применить**. Бот сам пропишет порты 80/443 в `docker-compose.yml` и перезапустится; сертификат подтянется при первом запросе.
3. Порты 80 и 443 на сервере должны быть свободны. Если их уже занял reverse-proxy панели (Caddy/nginx) — используйте способ B.

**Способ B — за reverse-proxy панели.** Бот отдаёт plain HTTP на `8080`, HTTPS терминирует прокси панели: добавьте в Caddy/nginx проксирование `/webhook/` → `http://remnabot:8080`. Либо опубликуйте порт (`ports: ["8080:8080"]`) и поставьте свой HTTPS-прокси.

Затем в кабинете провайдера укажите URL вебхука (домен из способа A или ваш прокси):

- **ЮKassa:** `https://ВАШ_ДОМЕН/webhook/yookassa`
- **CryptoBot:** `https://ВАШ_ДОМЕН/webhook/cryptobot`
- **Platega:** `https://ВАШ_ДОМЕН/webhook/platega`
- **Tribute:** `https://ВАШ_ДОМЕН/webhook/tribute` (тот же API-ключ, что в админке — по нему проверяется подпись)

Даже если вебхук не дойдёт, фоновый **реконсилятор** перепроверит статус у провайдера и довыдаст подписку — деньги не потеряются. Но вебхук всё равно стоит настроить ради мгновенной выдачи.

## 8. Переменные окружения

Задаются в `docker-compose.yml`. Обязательны только первые две — остальное обычно настраивается в мастере и менять не нужно.

| Переменная | Обяз. | Значение по умолчанию | Описание |
|---|:---:|---|---|
| `BOT_TOKEN` | ✅ | — | токен бота от @BotFather |
| `ADMIN_TELEGRAM_ID` | ✅ | — | ваш Telegram ID (число) — доступ в админку |
| `DATA_DIR` | | `/data` | каталог данных в контейнере (том `bot-data`) |
| `DB_KIND` | | — | `sqlite` / `postgres`; обычно задаёт мастер |
| `DATABASE_URL` | | — | DSN Postgres, если подключаете свою БД вручную |
| `SECRET_KEY` | | авто | ключ шифрования настроек; если пуст — генерируется в `/data/secret.key` |
| `PREMIUM_EMOJI` | | — | карта премиум-эмодзи в формате `ключ=id,ключ=id` |
| `COMPOSE_FILE_PATH` | | `/compose/docker-compose.yml` | путь к compose-файлу для самоменеджмента |
| `COMPOSE_PROJECT` | | `remnachillbot` | имя docker-compose проекта |
| `COMPOSE_HOST_DIR` | | `/opt/remnachillbot` | путь к папке бота **на хосте** |
| `PANEL_CONTAINER` | | `remnawave` | имя контейнера панели (для режима «Локально») |
| `PANEL_NETWORK` | | `remnawave-network` | docker-сеть панели (фолбэк, если контейнер не успел подняться). Официалка и eGames используют это имя — менять обычно не нужно |
| `SELF_CONTAINER` | | `remnabot` | имя контейнера самого бота |

> `SECRET_KEY` критичен: без него не прочитать зашифрованные настройки. Не теряйте файл `secret.key` из тома `bot-data`.

## 9. Обновление и бэкап

### Обновление

Самый простой путь — команда **`/update`** боту (скачает свежий образ и перезапустит). Либо вручную:

```bash
cd /opt/remnachillbot
docker compose pull
docker compose up -d
```

> Новый образ появляется в GHCR автоматически после каждого пуша в `main` (это делает CI).

### Бэкап

- **SQLite:** сохраните том `bot-data` целиком — там `bot.db` **и** `secret.key`.
- **PostgreSQL:**
  ```bash
  docker exec remnabot-db pg_dump -U remnabot remnabot > backup_$(date +%F).sql
  ```
  и дополнительно сохраните `secret.key` из тома `bot-data`.

Без `secret.key` восстановленная база будет нечитаемой (настройки зашифрованы) — храните его вместе с дампом.

## 10. Если что-то не работает

| Симптом | Что делать |
|---|---|
| Бот не стартует, в логах «BOT_TOKEN не задан» | не заменили `BOT_TOKEN`/`ADMIN_TELEGRAM_ID` в `docker-compose.yml` |
| «Нет связи с панелью» (Локально) | контейнер панели запущен и называется `remnawave`? Иначе задайте `PANEL_CONTAINER` |
| HTTP **401/403** от панели | неверный/просроченный API-token → создайте новый и пройдите `/setup` |
| Удалённо + eGames, соединение рвётся (**444**) | неверная кука, перепроверьте строку `ИМЯ=ЗНАЧЕНИЕ` |
| PostgreSQL не поднялся | бот остаётся на SQLite; проверьте, что проброшен `docker.sock` (он есть в compose), повторите выбор позже |
| Оплата прошла, подписка не выдалась сразу | проверьте вебхуки (раздел 6); реконсилятор довыдаст в течение
